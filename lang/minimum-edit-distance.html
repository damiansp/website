Minimum Edit Distance

Minimum edit distance is an algorithm that is used in a number of applications. In short, it is asking how different are two inputs? How different is string-1 from string-2? What is the shortest, simplest, or most likely transformation between string-1 and string-2?

One case it is used in is automatic spell-checking. For example, if a user types "I lov potat chips" or "I lov totato chips", the spell-checker can identify that "lov", "potat", and "totato" are not in its dictionary. Maybe "lov" was intended to be "Lev", "lava", "olive" or "love". Perhaps they meant "tomato" chips or "potato" chips. By computing a difference between the string and candidate options, costs can be assigned to each possibility, to determine which is the most "similar" to the input.

In brief, there are 3 types of edits that are allowed to transform between string-1 and string-3: insertions, deletions, or substitutions. To get from "lov" to "love" we insert an "e".

lov-
||||
love
---i

To get from "totato" to "tomato" we substitute an "m" for the second "t".

totato
||||||
tomato
--s---

There is then some cost associated with each type of edit. In the simplest case, the "Levenshtein Distance", insertions and deletions each have a cost of 1, and substitutions have a cost of 2 (equivalent to an insertion and a deletion). There is no cost when the sequences match.
